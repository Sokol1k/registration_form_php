Registration Form - PHP
Реализовать форму регистрации участников конференции.
Форма регистрации должна состоять из двух страниц:
1.	Страница с картой и формой.
    1.	Встроить карту (google map) должна располагаться сверху и растягиваться на всю ширину страницы. Высота карты должна быть 450px на десктопах.
    2.	На карте должен быть отмечен адрес: 7060 Hollywood Blvd, Los Angeles, CA
    3.	Под картой форма 1 с тайтлом “To participate in the conference, please fill out the form” и полями:
        1.	firstname *
        2.	lastname *
        3.	birthdate * (использовать какой то виджет/плагин для удобного выбора даты)
        4.	report subject *
        5.	country * (выпадающий список со странами)
        6.	phone * (поле с масками под ввод телефонов, например маска для американского формата имеет следующий вид “+1 (555) 555-5555”)
        7.	email * (поле должно быть уникальным)
    4.	Под формой справа находится кнопка Next, при нажатии на нее и успешной валидации данных в форме юзер переходит на форму 2.
    5.	Форма 2 содержит следующие поля:
        1.	company
        2.	position
        3.	about me (textarea)
        4.	photo (file field)
    6.	После нажатия на кнопку Next юзер должен видеть вместо формы кнопки социальных сетей: FB, TW, при нажатии на которые можно поделиться ссылкой на форму. Сообщение для расшаривания в TW: “Check out this Meetup with SoCal AngularJS! ”
    7.	Под тайтлом форм должна находится ссылка на страницу со списком всех участников - All members (178).
2.	Страница списка участников.
    1.	Отображать список участников в таблице с полями:
        1.	photo (если нету отображать какое нибудь изображение по умолчанию)
        2.	firstname + lastname в одной ячейке
        3.	report subject
        4.	email (ссылкой)
3.	Данные из форм должны сохранятся без перезагрузки страницы. При этом если на второй форме обновить страницу юзер должен снова попасть на эту же форму.
4.	Если юзер заполнил только первую форму все данные должны быть сохранены и он должен отображаться в списке All members.
5.	Адрес и текст шарилки для TW должен храниться в конфиге.
6.	Требования к технической части
    1.	Для верстки использовать Bootstrap 4.
    2.	База данных MySQL.
    3.	PHP без фреймворков  и сторонних библиотек.
    4.	JS - любые плагины, библиотеки и шаблонизаторы. Использовать возможности стандарта ES6
Для реализации задания использовать объектно ориентированный подход и обязательно реализовать MVC. Так же использовать namespace и PDO для работы с базой данных.
